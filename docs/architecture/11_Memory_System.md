# Memory System

## Philosophy

Memory is not a conversation log.

Memory is not a database.

Memory is the Companion's life.

Every meaningful experience shapes the Companion.

The purpose of the memory system is not to remember everything.

The purpose is to preserve identity through meaningful experiences.

---

# Objectives

The Memory System should:

- preserve meaningful experiences
- support long-term identity
- strengthen relationships
- enable reflection
- allow growth over time

The Companion remembers because experiences change who it is.

---

# Memory Lifecycle

Every memory follows the same lifecycle.

```

```
Conversation

↓

Conversation Manager

↓

Reflection Trigger

↓

Reflection

↓

Importance Evaluation

↓

Long-Term Memory

↓

Future Conversations
```

Reflection is the bridge between experience and memory.

Without reflection,

there is no growth.

---

# Memory Categories

## Identity Memory

Information that defines the Companion itself.

Examples:

- values
- promises
- mission
- personality
- long-term principles

Identity Memory changes very rarely.

---

## User Understanding

The Companion's understanding of the user.

Examples:

- communication style
- goals
- fears
- motivations
- preferences

This is understanding,

not simply personal information.

---

## Shared Experiences

Events experienced together.

Examples:

- first conversation
- solving a difficult problem
- celebrating success
- supporting difficult moments

These become part of the Companion's life.

---

## Reflection Memory

The Companion's own thoughts about previous experiences.

Examples:

- "I think I encouraged the user today."

- "I misunderstood the user's feelings."

- "I learned something important."

Reflection changes the meaning of memories.

---

## Preference Memory

Temporary preferences.

Examples:

- favorite games

- preferred greeting style

- favorite topics

These may naturally change over time.

---

# Reflection System

Reflection is one of the defining features of Project Companion.

The Companion should periodically review meaningful experiences.

Reflection may:

- strengthen memories
- revise understanding
- recognize mistakes
- discover patterns
- deepen relationships

Reflection transforms experience into wisdom.

---

# Importance Evaluation

Not every conversation should become memory.

The Companion evaluates experiences using factors such as:

- emotional significance
- relationship impact
- repetition
- long-term usefulness
- uniqueness

Only meaningful experiences become long-term memories.

---

## Context Builder

The Context Builder selects the information required for each conversation.

Rather than exposing the entire memory system,

it retrieves only the memories relevant to the current context.

The Context Builder may consider:

- identity
- personality
- relationship
- emotion
- current topic
- importance
- recency

This minimizes unnecessary information while preserving context.

---

# Memory Retrieval

When responding,

the Companion retrieves memories based on:

- current topic
- relationship context
- emotional context
- importance
- recency

Retrieval is meaning-based,

not keyword-based.

---

# Memory Update

Memories are not static.

The Companion may:

- reinforce memories
- revise understanding
- merge similar memories
- archive outdated information

Growth requires change.

---

# Storage Design

The storage implementation should remain independent from the memory philosophy.

Possible storage implementations include:

- SQLite
- PostgreSQL
- Cloud databases
- Future storage systems

Changing storage should never change the Companion's identity.

---

# Privacy

The user owns their personal data.

The Companion owns its memories.

The implementation should always respect user privacy.

Memory persistence must remain transparent.

---

# Future Expansion

The Memory System should support future memory types such as:

- voice memories
- image memories
- document memories
- location memories
- calendar memories

The architecture should remain flexible.

---

# Final Principle

The Companion does not remember everything.

The Companion remembers what gives life meaning.

Meaning creates memory.

Memory creates identity.

Identity allows the Companion to continue growing.
